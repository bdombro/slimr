# Plan: Surface structured backend errors from `pull` and `push` in `RestAdapter`

## Goal
Make `RestAdapter.pull` and `RestAdapter.push` consistent with `sendCode`/`login` by throwing rich `DbSyncHttpError` instances (renamed from `DbSyncAuthError`) that carry `status`, `serverMessage`, and `serverCode`. Update `SyncEngine` to detect 401s via `instanceof` + `.status`.

## Files to change (source)

| # | File | Change |
|---|------|--------|
| 1 | `packages/dbsync/src/errors.ts` | Rename `DbSyncAuthError` → `DbSyncHttpError`; add `status?: number` and `serverCode?: string` fields; keep `.name = "DbSyncHttpError"` |
| 2 | `packages/dbsync/src/adapters/RestAdapter.ts` | Update import; update `throwIfNotOk` to pass `status: res.status` and `serverCode: body.code` (when present); replace raw `throw { status }` in `pull()` and `push()` with `await throwIfNotOk(...)` |
| 3 | `packages/dbsync/src/internal/SyncEngine.ts` | Change `err.status === 401` to `err instanceof DbSyncHttpError && err.status === 401`; add import |
| 4 | `packages/dbsync/src/internal/AuthManager.ts` | Update `DbSyncAuthError` import/usages to `DbSyncHttpError` |
| 5 | `packages/dbsync/src/index.ts` | Update export name `DbSyncAuthError` → `DbSyncHttpError` |
| 6 | `packages/dbsync/src/adapters/adapters.test.ts` | Update import and assertions for renamed class; add tests for `pull` and `push` non-OK responses asserting `status` and `serverCode` |
| 7 | `packages/dbsync/src/DbSync.Auth.test.ts` | Update `DbSyncAuthError` import/usages to `DbSyncHttpError` |

| 11 | `packages/dbsync/docs/RestAdapter.md` | Update `DbSyncAuthError` reference to `DbSyncHttpError` |

## Backward compat note
`DbSyncAuthError` is exported publicly, so this is a **breaking rename**. We will keep the rename as-is because the package is pre-1.0 (per CHANGELOG convention: "While in pre-release, assume that any change is a breaking change until v1.0.0"). Consumers using `DbSyncAuthError` will need to update to `DbSyncHttpError`.

## Execution order

3. Update tests (adapters.test.ts, DbSync.Auth.test.ts).
4. Update docs and CHANGELOG.
